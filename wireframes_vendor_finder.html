Screen 1 — Sourcing Brief Intake Entry point · User describes what they're sourcing and sets criteria
rfp-vendor-finder.vercel.app
RFP Vendor Finder
How it works

Describe your sourcing need in plain language. The AI agent will search the web, identify vendors, and return a scored shortlist in under 5 minutes.

Describe what you're sourcing, the category, geography, and any key requirements.
e.g. "Cloud ERP software vendor for a mid-size manufacturing company in Southeast Asia, budget ~$500K. Must have local implementation support."
Select what matters most. These criteria will be used to score and rank each vendor. Adjust weights by clicking a tag.
Geographic coverage 25% ×
Industry specialisation 20% ×
Company size / stability 20% ×
Track record / references 20% ×
Pricing model 15% ×
+ Add criterion
UX note: Default criteria are pre-filled based on common procurement use cases. User can remove, add, or click any tag to adjust the weight. Weights auto-rebalance to 100% when a tag is removed. The sourcing brief is the only required field before the agent can run.

What's included in the export:
  • Sourcing brief (as entered)
  • Scoring criteria and weights
  • All 7 shortlisted vendors ranked by fit score
  • Per-vendor: name, website, score, AI rationale (2–3 sentences), source URLs
  • Disclaimer: AI-researched — verify before inviting to RFP
UX note: Export is a modal/panel over the results screen — user doesn't navigate away and lose context. CSV is the default as it's more likely to be used in a working procurement context (pasted into a deck or bid register). The disclaimer is included in every export to manage expectations about AI-generated data.
